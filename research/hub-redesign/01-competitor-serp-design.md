## 2. PAGE STRUCTURE OF THE TOP HUBS

Reverse-engineered from Money.com `/best-dental-insurance/` and MoneyGeek `/insurance/health/dental-insurance-costs/` (both fully fetched), cross-checked with SERP snippets for NerdWallet/Forbes/ValuePenguin.

**Standard winning skeleton (top to bottom):**
1. **H1 + one-sentence answer** ("The average cost… is $30/month") — answers the query in the first line for AI Overview/featured snippet capture.
2. **Author + Editor byline block** — name, credential (CFP, PhD, "Licensed Health Insurance Agent"), headshot, "Written by / Edited by", **"Updated: [date]"**, and an "Advertising & Editorial Disclosure" link.
3. **Sticky / inline Table of Contents** ("On This Page" jump links) — MoneyGeek shows it twice (top + repeated).
4. **Key Takeaways** card — 3 bulleted blue-check takeaways.
5. **Methodology block** (collapsible "Our Methodology" / "Why trust us?") — names the data source (e.g. CMS exchange data, "we analyzed 17 providers", "X plans scored on premium, coverage, annual max, waiting period").
6. **"Best for X" cards / ranked list** (editorial best-of pages): Best Overall, Best for Braces, Best for Value, Best for No Waiting Period, Best for Seniors, Best for Implants/Major Work, Best for Teeth Whitening. Each card = badge + monthly price + annual max + waiting period + pros/cons + CTA.
7. **Comparison tables** — the core asset. By State, by Plan Type (HMO/EPO/PPO/Indemnity), by Provider. Columns: monthly cost, annual cost, deductible, annual max, waiting period.
8. **Cost-of-procedures table** (cleaning $100, filling $148, root canal $1,000, crown $1,288, implant $3k–7k) — anchors "is insurance worth it".
9. **"How it works" + glossary** — premium, deductible, copay, coinsurance, annual maximum, MOOP — each as an icon card with a one-paragraph definition.
10. **Situational sections** — Job-based, Medicare, Medicaid, self-employed, students.
11. **FAQ** (3–6 Q&As, mirrors People Also Ask).
12. **Internal-link rail** ("Learn More About…") to sibling sub-pages + breadcrumb (Home › Insurance › Health › Dental).
13. **Author bio footer** + **Sources list** (KFF, HHS, CMS — outbound .gov/.org citations for E-E-A-T).

## 3. DESIGN / TRUST (E-E-A-T) PATTERNS THAT READ AS AUTHORITATIVE

- **Dated freshness in the H1/title** — "(2026 Guide)", "of June 2026", visible **"Updated: April 23, 2026"**. Freshness is a strong ranking correlate in this YMYL category.
- **Named, credentialed author + separate editor**, headshots, LinkedIn link, full bio block at the end. (Google YMYL signal.)
- **Explicit methodology** — "we collected data directly from CMS", "we analyzed coverage, annual maximum, premium, out-of-pocket, member experience". Removes "who says?" doubt.
- **Outbound authoritative citations** — KFF, HHS, CMS, ADA. A numbered Sources section.
- **Advertising/editorial disclosure** link (transparency = trust).
- **Comparison tables** with clean column headers and "Show more" expanders — scannable, screenshot-able, snippet-friendly.
- **Badges** ("Best for Value", "Best Overall", review-count/star rating) and **Key Takeaways** check-cards.
- **Icon-led glossary cards** instead of walls of text.
- Mobile-first jump-link TOC.

## 5. SCHEMA THE WINNERS USE

- **Article / NewsArticle** with `author`, `editor`, `datePublished`, `dateModified` (freshness) — universal on editorial hubs.
- **BreadcrumbList** — universal.
- **FAQPage** — on the FAQ block (drives PAA + rich results).
- **ItemList** — for "best of" ranked lists.
- **Table** markup / well-structured `<table>` for the comparison grids (snippet capture).
- **Review / AggregateRating** — on carrier review pages.
- **Organization / Person** (author bio) — supports E-E-A-T.
- Carrier pages add **Product / Offer / Service** schema.

**CoverCapy should ship:** Article (dated, authored) + BreadcrumbList + FAQPage + ItemList (best-for cards) + Table + Organization, and Review/AggregateRating on per-carrier sub-pages. (T5 dentist pages already use Dentist/LocalBusiness/MedicalOrganization — reuse that graph for directory cross-links.)

## 9. DESIGN DIRECTION (premium jade/cream concierge, NOT generic SaaS)

Keep CoverCapy's existing token system (`--teal-night #082A30`, `--teal-700`, `--mint #5BE0A0`, `--cream-card`, `--cream`, `--gold-soft`, Fraunces serif headlines, Inter Tight body). Translate every competitor pattern into the boutique-concierge aesthetic:

- **Byline/trust block** → cream card with serif name, small jade credential pill, "Updated" in `--ink-faint`. No stock-photo clutter.
- **Comparison tables** → cream rows on `--cream-card`, jade header row, `--mint-soft` for the "best" highlighted cell, `--gold-soft` for warning notes (e.g. "no annual max = DHMO only"). No zebra-stripe SaaS look, no gradients, no glassmorphism.
- **"Best for X" cards** → cream cards, serif title, one jade badge (no Capy Crowns in this context), price + annual max + waiting period as a tidy stat row, single mint CTA.
- **Glossary** → quiet icon cards on cream, serif term, body-color definition.
- **Methodology / Key Takeaways** → collapsible cream panel with a thin `--line` border, mint check marks.
- **CTAs** → "Verify my PPO — free" (mint button, jade text) and "Plans from $30/mo". Concierge voice; **no em-dashes, no roman numerals, no countdown timers, no member-ID storage** (per CLAUDE.md copy rules).
- Overall feel: luxury boutique-hotel concierge desk, not an affiliate comparison farm. The data density of MoneyGeek, dressed as a private members' guide.
