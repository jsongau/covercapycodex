## 3. Focus visibility

- No `:focus-visible` rule exists anywhere in the file (zero matches).
- Several interactive elements explicitly REMOVE the default outline with no replacement: `input[type=range]{outline:none}` L166, `.cmp-empty .ce-sel:focus{outline:none...}` L235, `.he-zip input:focus{outline:none...}` L362. These rely on a border-color change only, which is a color-only focus cue and is itself low contrast in the gold theme.
- `.omni-input:focus` (L573, L753, L784) does add a 3px box-shadow ring, which is acceptable, but the gold ring `rgba(184,146,79,.15)` is very faint.
- Buttons (`.btn-green`, `.chip`, `.goal`, `.tchip`, `.theme-tog button`, `.faq-openall`, `.pl-close`) have no defined focus style at all, so they fall back to the UA outline only where it has not been stripped.

Remediation: add a global `:focus-visible{outline:2px solid var(--green-d); outline-offset:2px}` and stop stripping outlines without a visible replacement.

## 4. Reduced motion

- No `@media (prefers-reduced-motion)` block exists (zero matches).
- The page uses `html{scroll-behavior:smooth}` (L63), many `transition:` declarations, the `.toc-prog` width transition (L119), and the `cmp` enter animations. None are gated for users who request reduced motion.

Remediation: add
```css
@media (prefers-reduced-motion: reduce){
  html{scroll-behavior:auto}
  *{animation-duration:.001ms !important; animation-iteration-count:1 !important; transition-duration:.001ms !important}
}
```

## 5. Minimum font sizes

Body is 16px (L64). Several supporting labels sit at 10-12px but none below 13px for actual body/reading prose. Smallest values found:
- 9px: `.mnav .ca` (L112) - decorative caret, acceptable.
- 10px: `.cmp-empty .ce-slot` L229, `.ft-mob-lbl` L331, `.plan-verified` L519, `.dc-rib`/`.dc-feat`/`.dc-tier`/`.dc-avb` L595/L600/L607/L604, `.omni-type` L587, `.pscore span` L540 - all uppercase micro-labels/badges, not body copy.
- 11px and 12px: numerous secondary labels.

No reading-level body text is under 13px, so the "no body under 13px" rule holds. The 10px uppercase labels are borderline for legibility; recommend lifting to 11px minimum where they carry meaning (for example `.plan-verified` L519 and `.ft-mob-lbl` L331).

## 6. Coverage never conveyed by color alone

Currently PASS by default: the plan comparison table (L954-961) and the per-card coverage lists (`.cov` L551, `.brief-cov` L701) use text values, not color swatches. The orphaned triad tokens (L53-55) are not yet applied. Guardrail for the redesign: any future coverage chip built on `--covered/--partial/--notcov` must pair the tint with both the `*-ink` text label and a glyph (for example a check, half-circle, dash) so the state survives grayscale and color-blind viewing.
