## 1. Pillar Definition

### What the Glossary Page Covers

The `/dental-insurance-glossary/` pillar page is the single authoritative reference for dental insurance terminology on CoverCapy. It functions as a living dictionary of ~60-80 defined terms, organized alphabetically with anchor links, covering:

- **Plan mechanics:** deductible, annual maximum, benefit period, coinsurance, copay, coordination of benefits, EOB (explanation of benefits), in-network vs. out-of-network, UCR (usual, customary, and reasonable), missing tooth clause, waiting period, rollover maximum
- **Coverage categories:** preventive, basic, major, orthodontic, implant, endodontic, periodontic
- **PPO-specific terms:** preferred provider, participating dentist, non-participating dentist, direct reimbursement, dual coverage, primary vs. secondary insurer
- **Procedure codes:** CDT code (definition only — individual CDT code pages are separate cluster pages), ADA code, fee schedule, table of allowances
- **Enrollment terms:** open enrollment, qualifying life event, COBRA dental, group vs. individual plan, dependent coverage, CHIP, DHMO vs. PPO vs. indemnity
- **Financial terms:** premium, out-of-pocket maximum, predetermination, pre-authorization, claim, assignment of benefits, balance billing

### What the Glossary Page Does NOT Cover

The pillar page deliberately excludes the following — these are owned by dedicated cluster or situation pages:

- How to choose a specific plan (owned by `/compare-ppo-dental-plans/`)
- Step-by-step "how dental insurance works" explainer (owned by gap page: `/how-dental-insurance-works/`)
- Situation-specific advice (between-jobs, self-employed, no-waiting-period — owned by existing situation pages)
- Individual CDT code deep-dives like D2740 or D7210 (owned by gap cluster pages)
- State-specific coverage rules (owned by `/dental/{state}/` hub pages)
- Dentist finder or insurance verification (owned by `/find-my-dentist/`)
- Braces and orthodontic cost breakdowns (owned by gap page: `/dental-insurance-for-braces/`)

The pillar establishes definitions. Cluster pages own application, context, and decisions.

## 4. Topic Authority Map

The content cluster collectively signals expertise on PPO dental insurance to Google through four authority signals:

### Breadth Signal (Glossary Pillar)
The glossary pillar demonstrates that CoverCapy understands the full landscape of dental insurance terminology. 60-80 defined terms across plan mechanics, coverage categories, CDT codes, and enrollment create a breadth signal: this site knows the entire domain.

### Depth Signal (Cluster Pages)
Each cluster page takes one or a handful of glossary terms and goes deep — worked examples, dollar amounts, edge cases, regional variation, and decision frameworks. Depth signals expertise beyond surface-level content.

### Entity Coverage (CDT Code Series)
CDT code pages build a structured entity graph around dental procedures. Google's Knowledge Graph treats CDT codes as structured entities. A site that defines and contextualizes 20+ CDT codes in a consistent format earns topical authority over dental procedure search queries.

### Situational Completeness (Situation Pages)
The four existing situation pages plus the gap pages cover every major buyer journey: new to insurance, between jobs, self-employed, needs braces, needs implants, hit annual maximum, confused by EOB. Covering the full decision space signals comprehensive resource status.

### Authority Signals by Google E-E-A-T Pillar

| E-E-A-T Pillar | How This Cluster Signals It |
|----------------|----------------------------|
| Experience | Worked dollar examples, real UCR ranges, annotated EOB diagram |
| Expertise | CDT code definitions, predetermination process, coordination of benefits mechanics |
| Authoritativeness | Breadth of coverage (60-80 terms), cross-linking between cluster pages, structured schema |
| Trust | Transparent about what plans do NOT cover, missing tooth clause content, no-spin EOB explainer |

## 5. Semantic Co-Occurrence

Semantic co-occurrence means placing related terms together on the same page so Google's language model learns which concepts belong in the same semantic neighborhood. Below is the target co-occurrence map by page type.

### Glossary Pillar — Must Co-Occur Everywhere
These term pairs must appear in proximity on the pillar page because they are semantically bonded in Google's dental insurance model:

| Term A | Term B | Why They Co-Occur |
|--------|--------|-------------------|
| deductible | annual maximum | Both are dollar caps; buyers confuse them |
| coinsurance | UCR | UCR is the base on which coinsurance is calculated |
| waiting period | qualifying life event | QLE is how you bypass waiting periods |
| in-network | fee schedule | In-network pricing is defined by fee schedule |
| missing tooth clause | implant | Missing tooth clause most commonly affects implants |
| predetermination | major services | Predetermination is required for major work |
| EOB | claim | EOB is what you get after a claim is filed |
| COBRA dental | deductible reset | COBRA timing is driven by deductible reset concerns |
| CDT code | ADA code | Synonymous; searchers use both terms |
| preventive / basic / major | coinsurance | Coverage tiers are defined by coinsurance percentages |

### `/how-dental-insurance-works/` — Core Mechanics Cluster
Must appear together: premium, deductible, coinsurance, annual maximum, preventive, claim, EOB. Supporting: benefit period, assignment of benefits, in-network.

### `/what-is-a-ppo-dental-plan/` — Plan Type Cluster
Must appear together: PPO, DHMO, indemnity, preferred provider, in-network, out-of-network, UCR, fee schedule. Supporting: network, coinsurance, deductible.

### `/dental-insurance-for-implants/` — Implant Coverage Cluster
Must appear together: implant, major services, missing tooth clause, annual maximum, predetermination, bone graft, waiting period. Supporting: coinsurance, balance billing, out-of-network.

### `/dental-insurance-for-braces/` — Orthodontic Cluster
Must appear together: orthodontic, lifetime orthodontic maximum, waiting period, dependent coverage, major services. Supporting: Invisalign, coinsurance, annual maximum.

### CDT Code Pages — Procedure Cluster
Each page must include: CDT code, ADA code, procedure name, fee schedule, UCR, coverage category (preventive/basic/major), coinsurance, predetermination (if applicable). Supporting: claim, EOB, in-network rate.

### `/understanding-your-dental-eob/` — EOB Cluster
Must appear together: EOB, claim, coinsurance, deductible, UCR, balance billing, assignment of benefits, in-network, out-of-network. Supporting: appeal, coordination of benefits.

### Link Equity Rules

**Pillar receives from:**
- All cluster pages (each links back to glossary at least once in body copy)
- All T5 dentist profile pages (link from "About" section or network terminology)
- External backlinks (target: dental associations, personal finance sites, employee benefits blogs)

**Pillar passes to:**
- Situation pages (high priority — these convert)
- `/compare-ppo-dental-plans/` (primary conversion CTA, appears once in pillar body)
- Gap pages (contextual inline links from glossary term definitions)

**Cluster pages pass to each other (lateral equity):**
- `/how-dental-insurance-works/` links to `/dental-insurance-deductible-explained/`, `/what-is-a-ppo-dental-plan/`
- `/dental-insurance-for-implants/` links to `/dental-insurance-missing-tooth-clause/`, `/dental-insurance-deductible-explained/`
- CDT code pages cross-link to each other (e.g., D2740 crown page links to D2950 core buildup page)
- All cluster pages link to `/compare-ppo-dental-plans/` as the primary conversion CTA

**Location pages pass to pillar:**
- T5 pages link to glossary for terms like "PPO network," "in-network," "coinsurance" where these appear in the About prose block
- State hub pages link to glossary in their "Understanding dental coverage in [State]" editorial section

**Do NOT:**
- Have glossary link to all 6,400 location pages (avoid diluting equity with non-topical links)
- Have situation pages link to other situation pages (keeps cluster hierarchy clean)
- Have CDT code pages link to situation pages (they are different sub-clusters)
