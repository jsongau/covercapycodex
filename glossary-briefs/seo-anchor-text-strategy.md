# SEO Anchor Text Strategy — covercapy.com/dental-insurance-glossary/
## CoverCapy | Updated June 2026

This document covers all anchor text decisions for links TO and FROM the dental insurance glossary page. Follow these rules across every page type to build topical authority without triggering over-optimization signals.

---

## 1. Links TO the Glossary — by Source Page Type

The glossary lives at `https://covercapy.com/dental-insurance-glossary/`. Every link pointing to it should use varied, contextual anchor text. The page slug itself ("dental-insurance-glossary") is already keyword-rich, so anchors should lean descriptive and conversational rather than repeating the exact URL slug.

### 1a. Homepage (`/index.html`)

Limit: 1 link to the glossary from the homepage. Place it in an educational context near any mention of PPO terms or plan comparison copy.

**Recommended placement:** In a section explaining how PPO coverage works, after a mention of terms like "deductible" or "annual maximum."

```html
<a href="/dental-insurance-glossary/" class="cc-prose-link">
  what these terms actually mean
</a>
```

**Fallback anchor (rotate on A/B test):**
```html
<a href="/dental-insurance-glossary/" class="cc-prose-link">
  plain-language PPO glossary
</a>
```

Do NOT use "dental insurance glossary" as the anchor text on the homepage. That anchor is too exact-match and should be reserved for the footer nav label only.

**Tooltip "Full glossary" link (current):**

See Section 6 for the tooltip link recommendation. Keep or replace this — guidance is in Section 6.

### 1c. Find My Dentist (`/find-my-dentist.html`)

### 1d. T5 Dentist Profile Pages (generated by `seo-build/generate-plans.js`)

Each T5 page should carry exactly 1 link to the glossary, placed in the "About" prose section or inside the FAQ accordion. The anchor text should reference the dentist's accepted carriers or a relevant term.

**Recommended anchor text pattern (vary by page, use one per profile):**

Option A (deductible hook):
```html
<a href="/dental-insurance-glossary/#term-deductible" class="cc-prose-link">
  how your deductible works with PPO coverage
</a>
```

Option B (in-network hook):
```html
<a href="/dental-insurance-glossary/#term-in-network" class="cc-prose-link">
  staying in-network at this office
</a>
```

Option C (annual maximum hook):
```html
<a href="/dental-insurance-glossary/#term-annual-maximum" class="cc-prose-link">
  your annual coverage limit explained
</a>
```

Option D (generic, use when no specific term fits):
```html
<a href="/dental-insurance-glossary/" class="cc-prose-link">
  a plain-language guide to PPO dental benefits
</a>
```

**Implementation in generate-plans.js:**

Add a helper to the generator and use it inside `buildDentistPage()`. Rotate among options using the dentist's index position or a hash of their slug to ensure variety across the 6,400 pages.

```javascript
const GLOSS_BASE = 'https://covercapy.com/dental-insurance-glossary';

function glossAnchor(d, idx) {
  const opts = [
    { href: `${GLOSS_BASE}/#term-deductible`,    text: 'how your deductible works with PPO coverage' },
    { href: `${GLOSS_BASE}/#term-in-network`,     text: 'staying in-network at this office' },
    { href: `${GLOSS_BASE}/#term-annual-maximum`, text: 'your annual coverage limit explained' },
    { href: `${GLOSS_BASE}/`,                     text: 'a plain-language guide to PPO dental benefits' },
  ];
  return opts[idx % opts.length];
}
```

## 2. Links FROM the Glossary to Other CoverCapy Pages

Each outbound link from the glossary page should serve a conversion goal: move a reader from understanding a term to acting on it. The glossary should link to 3 primary destination types: the compare page, the find-dentist page, and the individual T5 profile pages.

### 2a. Links to the Compare Plans Page

Place this CTA once at the top of the glossary page (above the term list) and once at the bottom (after the last term).

**Top CTA:**
```html
<a href="/compare-ppo-dental-plans/" class="cc-btn-primary">
  Compare PPO plans now
</a>
```

**Bottom CTA:**
```html
<a href="/compare-ppo-dental-plans/" class="cc-btn-secondary">
  Ready to pick a plan? Compare your options.
</a>
```

Anchor text rationale: "Compare PPO plans" is the primary conversion keyword for that page. Do not use "dental insurance" alone or "compare dental insurance" -- those are too broad and dilute the page's PPO-focused targeting.

### 2b. Links to the Find My Dentist Page

Place 1 link inside or after the definition of "in-network" and 1 link in any sidebar or sticky rail:

**Inside the "in-network" term definition:**
```html
<a href="/find-my-dentist/" class="cc-prose-link">
  find in-network PPO dentists near you
</a>
```

**Sticky rail or end-of-page CTA:**
```html

  Find a PPO dentist in your area
</a>
```

### 2c. Links to T5 Dentist Profiles

Do not link from the glossary page to individual T5 profiles. That creates millions of potential outbound link combinations and dilutes the page. Instead, link to the find-my-dentist search as the conversion path.

### 2d. Links to the No-Waiting-Period Landing Page (if it exists)

Inside the "waiting-period" term definition:

## 4. Avoiding Over-Optimization

Over-optimization risk is highest when:
- The same anchor text appears on more than 3 pages pointing to the same URL
- Anchor text exactly matches the target page's title or H1
- Keyword-heavy anchors cluster in the same page section

### Terms to rotate across pages

**"dental insurance glossary"** -- use ONLY in the footer nav label. Do not use this phrase as anchor text anywhere else on the site. It exactly matches the page title pattern and will look unnatural if it appears repeatedly.

**"PPO glossary"** -- maximum 3 uses sitewide. Acceptable on the compare page, homepage, and one hub page.

**"PPO dental benefits explained"** -- limit to 2 uses total.

**"annual maximum"** -- safe to use frequently as contextual anchor (the term appears naturally in plan-related copy), but vary the surrounding sentence structure so the links are not templated.

**"deductible"** -- same as annual maximum. Safe as contextual anchor. The risk is low because this word appears in genuine editorial contexts.

### Rotation framework

For the 6,400 T5 dentist pages, use a rotation system (see Section 1d implementation above). Do not use the same anchor text more than once per 4 consecutive pages. A simple modulo-4 rotation ensures variety and prevents any one anchor from dominating the link profile.

## 5. Brand vs. Keyword Anchor Ratio

### Recommended ratio for links pointing TO the glossary

| Anchor Type | Target Share | Examples |
|---|---|---|
| Contextual keyword | 55% | "how your deductible works", "in-network allowed amount", "your annual coverage limit explained" |
| Partial keyword | 20% | "PPO terms explained", "plain-language PPO glossary", "how PPO dental insurance works" |
| Branded | 15% | "CoverCapy's PPO glossary", "CoverCapy explains" |
| Navigational / generic | 10% | "learn more", "see all terms", the footer label |

### Notes on branded anchors

Branded anchors reinforce brand entity signals. Use them on T5 pages with phrasing like:

```html
<a href="/dental-insurance-glossary/" class="cc-prose-link">
  CoverCapy's plain-language PPO glossary
</a>
```

Do not overuse "CoverCapy" in anchor text. The brand is already established via the domain, logo links, and schema. Branded anchors beyond 20% of the total start to look self-promotional rather than editorial.

### Ratio for links pointing FROM the glossary

All outbound links from the glossary to CoverCapy pages should use conversion-oriented anchors, not generic text:

| Anchor Type | Target Share | Examples |
|---|---|---|
| Action-oriented | 60% | "Compare PPO plans now", "Find in-network PPO dentists near you" |
| Contextual within definition | 30% | "PPO plans with no waiting period", "in-network allowed amount" |
| Generic | 10% | "learn more" (only for supplemental references) |

Never use "click here" or "read more" as anchor text anywhere on the site.

## 6. Tooltip Link Text Recommendation

### Current tooltip "Full glossary" link

The compare page tooltip engine currently shows:
```html
<a class="cc-tooltip-link">Full glossary</a>
```

This link points to the term's anchor within the glossary (`#term-{slug}`). The text "Full glossary" is functional but generic.

### Recommendation: Change it

"Full glossary" is weak anchor text for two reasons:
1. It gives Google no signal about what kind of content the destination contains.
2. It does not reassure the user that clicking will give them more of what they need right now.

### Recommended replacement options

**Option A — Most informative:**
```html
<a class="cc-tooltip-link">See full definition</a>
```

Rationale: mirrors the user's intent (they want more than the tooltip summary), is natural language, and works regardless of which term triggered the tooltip.

**Option B — Brand-forward:**
```html
<a class="cc-tooltip-link">More in the PPO glossary</a>
```

Rationale: signals the destination is a glossary, reinforces the PPO brand positioning, and is specific enough to be useful anchor text.

**Option C — Conversion-forward:**
```html
<a class="cc-tooltip-link">Full explanation + examples</a>
```

Rationale: sets expectation of richer content (examples, real-world scenarios) which may improve click-through.

### Final call

Use **Option A ("See full definition")** as the default. It is the most universally accurate, works for every term in the GLOSS array, and avoids over-optimizing a tooltip link that appears up to 21 times on the compare page.

Update the tooltip JS in `compare-ppo-dental-plans.html`:

```javascript
// Current (line ~2060):
<a class="cc-tooltip-link">Full glossary</a>

// Replace with:
<a class="cc-tooltip-link">See full definition</a>
```
