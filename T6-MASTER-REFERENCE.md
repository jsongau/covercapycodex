## WHAT T6 IS

T6 pages are **individual doctor profiles** — the person, not the practice.

- T5 = `Abri Dental Beverly Hills` (LocalBusiness / practice entity)
- T6 = `Dr. David Abri, DDS` (Person + Physician entity)

A doctor can work at multiple T5 offices. A T5 office can have multiple T6 doctors. Many-to-many via `provider_offices` junction table in Supabase.

## URL ARCHITECTURE

```
https://covercapy.com/dentists/{state-slug}/{doctor-slug}/
```

Examples:
```
https://covercapy.com/dentists/california/david-abri/
https://covercapy.com/dentists/california/isaac-sun/
https://covercapy.com/dentists/texas/frank-curry/
```

**Slug derivation:**
```javascript
// Strip "Dr." prefix and credential suffix
"Dr. David Abri, DDS"  → "david-abri"
"Dr. Isaac Sun DDS"    → "isaac-sun"
"Afarin Anbarani, DDS" → "afarin-anbarani"
```

Source column: `provider_profiles.slug` (canonical, unique per state).
State slug: from `provider_profiles.license_state` (dental licenses are state-issued).
NEVER use city in T6 URL — doctors practice in multiple cities.

**Collision resolution:** if two "David Lee" in California, append primary city: `david-lee-beverly-hills`. Never append `-2` or `-dds`.

**File output:**
```
/dentists/{stSlug}/{docSlug}/index.html
```

### Nav + Footer (dynamic fetch, identical to find-my-dentist.html)

```html
<div id="cc-nav-mount"></div>
<script>
(function(){
  function mount(url, id, after){
    fetch(url).then(r=>r.text()).then(html=>{
      var el=document.getElementById(id);
      if(el){ el.innerHTML=html; }
      // replay any <script> tags inside the fetched HTML
      el.querySelectorAll('script').forEach(function(s){
        var n=document.createElement('script');
        if(s.src){ n.src=s.src; n.defer=true; }
        else { n.textContent=s.textContent; }
        document.head.appendChild(n);
      });
      if(after){ after(); }
    }).catch(function(){});
  }
  mount('/components/mega-nav.html','cc-nav-mount',function(){
    var s=document.createElement('script');
    s.src='/assets/js/mega-nav.js';
    s.defer=true;
    document.body.appendChild(s);
  });
})();
</script>

<!-- ... page body ... -->

<div id="cc-footer-mount" class="cc-footer-mount"></div>
<script>
(function(){
  function mount(url,id,after){
    fetch(url).then(r=>r.text()).then(html=>{
      var el=document.getElementById(id);
      if(el){ el.innerHTML=html; }
      if(after){ after(); }
    }).catch(function(){});
  }
  mount('/components/footer.html','cc-footer-mount',function(){

```

**CRITICAL:** After the footer `</script>` closing tag, the NEXT script block must be a SINGLE `<script>` tag. A `<script><script>` double-open breaks all page JS. This was the bug that caused "Loading profile..." on all 1,148 pages in June 2026. It has been fixed in the generator and all live pages.

14. Nearby doctors — rail of other T6 profiles in same state

## SEO — CRITICAL RULES

### Title tag formula

```
Dr. {Full Name}, {Credential} | {Primary Specialty} {Primary City} {State Abbr} | CoverCapy
```

Examples:
```
Dr. Isaac Sun, DDS | General Dentist Fountain Valley CA | CoverCapy
Dr. David Abri, DDS | Cosmetic Dentist Beverly Hills CA | CoverCapy
Dr. Jane Lee, DMD | PPO Dentist San Diego CA | CoverCapy
```

Max 60 characters. Truncate city (never name). Use state abbreviation (2 letters), not full state name.

### Meta description formula

```
Dr. {Last} practices {specialty} at {Practice} in {City1}{City2 if exists}, {StateAbbr}.
Accepts {Carrier1} and {Carrier2} PPO. Verify your coverage free.
```

Max 155 characters. Must include: specialty + practice name + city + 2 carriers + verify CTA.

### Indexability gate

Pages are `noindex` when:
- `!hasIdent` — no NPI and no license number

That's the ONLY gate. Office count and bio word count are NOT gates.

### Internal linking (critical — T6 pages have no PageRank without this)

Every T5 page that has a linked doctor must link to that doctor's T6 page. This is the primary source of T6 inbound links. Without T5→T6 links, Google deprioritizes T6 pages (too deep, no inbound PageRank).

## T5 ↔ T6 CROSS-LINKING — How to Co-Edit T5 Pages

This is the critical integration. T6 pages have no inbound PageRank unless T5 pages link to them.

### How it works at build time

`generate-providers.js` writes a side-effect file after building all T6 pages:

```
/dentists/.cross-links.json
```

Format:
```json
{
  "abri-dental-beverly-hills": {
    "provSlug": "david-abri",
    "provName": "Dr. David Abri, DDS",
    "stSlug": "california"
  },
  "kyt-dental-services": {
    "provSlug": "isaac-sun",
    "provName": "Dr. Isaac Sun, DDS",
    "stSlug": "california"
  }
}
```

**Key:** office `dentists.slug`. **Value:** the doctor who works there.

### Changes needed in generate-plans.js (T5 generator)

At the top of `generate-plans.js`, load the cross-links map:

```javascript
let crossLinks = {};
try {
  crossLinks = JSON.parse(
    fs.readFileSync(path.join(OUT_DIR, 'dentists', '.cross-links.json'), 'utf8')
  );
} catch (e) { /* no T6 build yet — fine, skip linking */ }
```

Inside `buildDentistPage(d, market, state)`, replace the plain `doctor_name` text with a linked version:

```javascript
// BEFORE (current):
${d.doctor_name ? `<p class="hero-doctor">Led by ${esc(d.doctor_name)}</p>` : ''}

// AFTER (with T6 link):
const provData = crossLinks[d.slug];
const doctorLine = d.doctor_name
  ? provData
    ? `Led by <a href="${BASE_URL}/dentists/${provData.stSlug}/${provData.provSlug}/"
         class="hero-doctor-link">${esc(d.doctor_name)}</a>`
    : `Led by ${esc(d.doctor_name)}`
  : '';
// Then use ${doctorLine} in the template
```

Also add to the T5 About section prose:
```javascript
// In the about paragraph, after practice name mention:
${provData
  ? `<a href="${BASE_URL}/dentists/${provData.stSlug}/${provData.provSlug}/">
     ${esc(d.doctor_name)}</a>`
  : esc(d.doctor_name)
}
```

And add to T5 JSON-LD `Dentist` schema node:
```json
"employee": {
  "@type": "Physician",
  "@id": "https://covercapy.com/dentists/{stSlug}/{provSlug}/#person",
  "name": "Dr. David Abri, DDS"
}
```

## CONTENT TIERS

### Tier 1 — NPI match only (no scraped bio)

Generated bio template: 130-160 words. 3 FAQs. noindex until license or NPI verified.

### Tier 2 — NPI + scraped bio

Custom bio: 250-350 words. 6-8 FAQs. index, follow.

### Tier 3 — Claimed profile

Full bio: 400-800 words. 10 FAQs. Spanish FAQ block if languages includes Spanish.

## BIO VOICE RULES

Never use:
- "passionate about dentistry"
- "gentle and caring"
- "state-of-the-art" (name the actual tech: iTero, CBCT, CEREC)
- "our team" (person profile — always "Dr. [Name]")
- "comprehensive care"
- "we strive to"
- em-dashes

Always:
- Specific over generic ("ceramic layering" beats "advanced techniques")
- Patient outcomes over capabilities
- Place-names ("grew up in Alhambra", "returned to Southern California")
- Name the carriers the office accepts
- First item in "Why Consider" = PPO verification (CoverCapy's core value)

## FAQ TEMPLATES (core 10)

1. Does Dr. {Name} accept {Carrier} dental insurance?
2. Where does Dr. {Name} practice?
3. What is Dr. {Name}'s NPI number? *(only if NPI populated)*
4. Is Dr. {Name} accepting new patients?
5. What procedures does Dr. {Name} offer?
6. What dental school did Dr. {Name} attend?
7. Does Dr. {Name} speak languages other than English? *(only if languages > 1)*
8. How do I verify my PPO insurance before seeing Dr. {Name}?
9. How long has Dr. {Name} been practicing?
10. What makes Dr. {Name} different from other dentists in {City}?

Tier 1 uses FAQs 1-3 only. All 10 for Tier 2+.

**Spanish FAQ block:** Generate when `provider.languages` includes 'Spanish'. 5 questions in Spanish. Insert after English FAQs. Add `lang="es"` to section.

## KNOWN ISSUES / WATCH OUT FOR

1. **Double script tag bug (FIXED June 2026):** After the footer mount `</script>`, the next script block must be a single `<script>` tag. A `<script><script>` double-open breaks all page JS — `boot()` never runs, page shows "Loading profile..." forever. Fixed in generator at line 922 and applied to all 1,148 live pages.

2. **Git divergence (June 2026):** Local and remote `main` diverged due to force pushes. Fix: `git merge --abort && git push origin main --force` from repo root on Mac Terminal.

3. **Generator must run from repo root.** `cd seo-build && node generate-providers.js` writes output to `seo-build/dentists/` which is gitignored. Always: `cd repo-root && node seo-build/generate-providers.js`.

4. **`seo-build/` is gitignored.** Generator changes are never committed. Only `/dentists/` output goes to git.

5. **`seo_path` is stale** in `dentists` table — has `/dentists/` prefix. Never use it. Always build T5 URLs from `stSlug + mkSlug + citySlug + d.slug`.

6. **Linux sandbox cannot push to GitHub.** Run git push from Mac Terminal with credentials.

7. **Never store member IDs.** Only `member_id_provided: boolean`.

8. **Strip UTM params** from all website URLs: `d.website.split('?')[0]`.

9. **T6 pages are standalone HTML.** They do NOT share JS with find-my-dentist.html or T5 pages. Nav and footer are fetched via `fetch('/components/mega-nav.html')` and `fetch('/components/footer.html')`.

10. **Run T6 generator before T5 generator** so `.cross-links.json` exists when T5 build reads it.
